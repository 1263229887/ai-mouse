# 电脑录音技术方案

## 方案概述

本方案使用电脑自带的麦克风进行录音，通过 Web Audio API (RecorderCore) 采集音频数据，经 WebSocket 发送到服务端进行语音识别。

## 音频数据格式

| 参数 | 值 |
|------|-----|
| 格式 | PCM (原始音频) |
| 采样率 | 16000 Hz |
| 位深度 | 16 bit |
| 声道数 | 1 (单声道) |
| 发送分块大小 | 1920 samples (120ms) |

## 核心文件

### 1. 主进程 SDK 封装
**文件**: `src/main/mouseSDK.js`

```javascript
// 电脑录音模式：禁用SDK默认录音行为
SDK_setVoiceKey(0)

// 鼠标录音模式：启用SDK录音（4键长按）
SDK_setVoiceKey(102)
```

### 2. 主进程录音控制
**文件**: `src/main/index.js`

```javascript
// 录音模式切换
let recordingSource = 'mouse' // 'mouse' 或 'computer'

// 电脑录音模式下，按键事件处理
if (type === 'deviceKeyEvent' && index === 102) {
  if (recordingSource === 'computer') {
    if (status === 1) startComputerRecording()   // 按下
    else if (status === 0) stopComputerRecording() // 松开
  }
}

// 启动电脑录音
function startComputerRecording() {
  // 创建弹窗，发送 useHardwareRecording: false
  popupWindow.webContents.send('start-speech-recognition', {
    wsUrl,
    extraParams,
    useHardwareRecording: false // 标记使用电脑录音
  })
}
```

### 3. 渲染进程音频处理
**文件**: `src/renderer/src/utils/speechRecognition.js`

```javascript
// 电脑录音模式
async connect(wsUrl, options = {}) {
  this.useHardwareRecording = options.useHardwareRecording || false
  
  if (!this.useHardwareRecording) {
    // 电脑录音模式：使用 RecorderCore
    this.recorder = new RecorderCore({
      type: 'pcm',
      bitRate: 16,
      sampleRate: 16000,
      onProcess: (buffers, powerLevel, ..., resampledData) => {
        if (resampledData && resampledData.length > 0) {
          this.handleAudioData(resampledData, powerLevel)
        }
      }
    })
    
    // 打开麦克风
    await this.recorder.open()
  }
}
```

## 录音流程

### 电脑录音模式

1. **用户选择电脑录音模式**
   - 调用 `setVoiceKey(0)` 禁用SDK默认录音行为
   - 录音模式标记为 `recordingSource = 'computer'`

2. **按下4键长按** (index=102, status=1)
   - `handleMouseMessage()` 检测到录音键按下
   - `startComputerRecording()` 设置 `isRecording = true`
   - 创建弹窗，发送 `useHardwareRecording: false`
   - `speechRecognition.js` 打开电脑麦克风并连接 WebSocket

3. **音频数据流转**
   - RecorderCore 采集电脑麦克风音频
   - `handleAudioData()` 处理并分块
   - WebSocket 发送 PCM 数据

4. **松开4键** (index=102, status=0)
   - `stopComputerRecording()` 设置 `isRecording = false`
   - `recorder.stop()` 停止录音器
   - 发送剩余音频数据
   - 发送 `is_speaking: false` 结束消息
   - 等待服务端返回 `is_final: true`
   - 执行粘贴操作并关闭弹窗

## 与鼠标硬件录音模式的区别

| 特性 | 电脑录音 | 鼠标硬件录音 |
|------|---------|-------------|
| 麦克风来源 | 电脑麦克风 | AI鼠标内置麦克风 |
| 音频采集 | RecorderCore (Web Audio API) | SDK OnDeviceAudioDataCallback |
| 权限要求 | 需要麦克风权限 | 无需浏览器权限 |
| 数据流向 | 渲染进程直接采集 | DLL → 主进程 → 渲染进程 |
| useHardwareRecording 标志 | false | true |
| setVoiceKey 设置 | setVoiceKey(0) 禁用 | setVoiceKey(102) 启用 |

## 模式切换逻辑

```javascript
// Home.vue 中的录音源切换
const recordingSource = ref('mouse') // 默认鼠标录音

function switchRecordingSource(source) {
  recordingSource.value = source
  // 通知主进程切换模式
  window.electron.ipcRenderer.send('switch-recording-source', source)
}

// 主进程处理模式切换
ipcMain.on('switch-recording-source', (event, source) => {
  recordingSource = source
  if (source === 'computer') {
    // 电脑录音模式：禁用SDK默认录音
    mouseSDK.setVoiceKey(0)
  } else {
    // 鼠标录音模式：启用SDK录音
    mouseSDK.setVoiceKey(102)
  }
})
```

## 注意事项

1. **模式切换时必须重置语音键**：切换到电脑录音模式时调用 `setVoiceKey(0)` 禁用SDK默认行为
2. **权限请求**：电脑录音模式需要用户授权麦克风权限
3. **按键事件处理**：无论哪种模式，都使用4键长按(index=102)控制录音开始/停止
4. **长按松开交互**：两种模式都采用长按录音、松开结束的交互方式
