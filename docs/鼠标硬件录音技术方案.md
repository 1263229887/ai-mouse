# AI鼠标硬件录音技术方案

## 方案概述

本方案使用AI鼠标设备自带的麦克风进行录音，通过SDK获取PCM音频数据，经IPC通信转发到渲染进程，再通过WebSocket发送到服务端进行语音识别。

## 音频数据格式

| 参数 | 值 |
|------|-----|
| 格式 | PCM (原始音频) |
| 采样率 | 16000 Hz |
| 位深度 | 16 bit |
| 声道数 | 1 (单声道) |
| 每帧字节数 | ~1010 字节 |
| 发送分块大小 | 1920 samples (120ms) |

## 核心文件

### 1. 主进程 SDK 封装
**文件**: `src/main/mouseSDK.js`

```javascript
// 关键配置
SDK_setVoiceKey(102)  // 设置语音键为4键长按

// 音频数据回调
const onDeviceAudioData = (deviceId, data, length) => {
  const audioData = koffi.decode(data, 'unsigned char', length)
  onDeviceAudioDataCallback?.(deviceId, audioData, length)
}
```

### 2. 主进程录音控制
**文件**: `src/main/index.js`

```javascript
// 处理设备音频数据
function handleDeviceAudioData(deviceId, audioData, length) {
  if (!isRecording) return
  
  // 转发到渲染进程
  const buffer = Buffer.from(audioData)
  popupWindow.webContents.send('device-audio-data', { 
    data: buffer,
    length: length
  })
}

// 按键事件处理 (index=102 为4键长按)
if (type === 'deviceKeyEvent' && index === 102) {
  if (status === 1) startRecording()   // 按下
  else if (status === 0) stopRecording() // 松开
}
```

### 3. Preload API 暴露
**文件**: `src/preload/index.js`

```javascript
onDeviceAudioData: (callback) => {
  const listener = (event, { data, length }) => {
    callback(data, length)
  }
  ipcRenderer.on('device-audio-data', listener)
  return () => ipcRenderer.removeListener('device-audio-data', listener)
}
```

### 4. 渲染进程音频处理
**文件**: `src/renderer/src/utils/speechRecognition.js`

```javascript
// 设置硬件音频监听器
setupHardwareAudioListener() {
  this.removeAudioDataListener = window.api.onDeviceAudioData((data, length) => {
    this.handleHardwareAudioData(data, length)
  })
}

// 处理硬件音频数据
handleHardwareAudioData(data, length) {
  // IPC 传输时 Buffer 会被序列化为 { type: 'Buffer', data: [...] }
  if (data && data.type === 'Buffer' && Array.isArray(data.data)) {
    rawBytes = new Uint8Array(data.data)
    pcmData = new Int16Array(rawBytes.buffer)
  }
  
  // 分块发送到 WebSocket
  while (this.sampleBuffer.length >= this.chunkSize) {
    const sendBuffer = this.sampleBuffer.slice(0, this.chunkSize)
    this.sendAudio(sendBuffer)
  }
}
```

## 录音流程

1. **按下4键长按** (index=102, status=1)
   - `handleMouseMessage()` 检测到录音键按下
   - `startRecording()` 设置 `isRecording = true`
   - 创建弹窗，发送 `useHardwareRecording: true`
   - `speechRecognition.js` 连接 WebSocket 并设置硬件音频监听器

2. **音频数据流转**
   - DLL 触发 `OnDeviceAudioDataCallback`
   - `mouseSDK.js` 解码数据并触发回调
   - `index.js` 通过 IPC 发送 `device-audio-data`
   - `speechRecognition.js` 接收、缓冲、分块发送到 WebSocket

3. **松开4键** (index=102, status=0)
   - `stopRecording()` 设置 `isRecording = false`
   - 发送剩余音频数据
   - 发送 `is_speaking: false` 结束消息
   - 等待服务端返回 `is_final: true`
   - 执行粘贴操作并关闭弹窗

## 与电脑录音模式的区别

| 特性 | 鼠标硬件录音 | 电脑录音 |
|------|-------------|---------|
| 麦克风来源 | AI鼠标内置麦克风 | 电脑麦克风 |
| 音频采集 | SDK OnDeviceAudioDataCallback | RecorderCore (Web Audio API) |
| 权限要求 | 无需浏览器权限 | 需要麦克风权限 |
| 数据流向 | DLL → 主进程 → 渲染进程 | 渲染进程直接采集 |
| useHardwareRecording 标志 | true | false |

## 注意事项

1. **语音键设置必须正确**：初始化时调用 `SDK_setVoiceKey(102)` 绑定4键长按
2. **回调函数原型必须是模块级变量**：避免被 GC 回收导致连接失败
3. **IPC 数据序列化**：Buffer 会被序列化为 `{ type: 'Buffer', data: [...] }`，需正确解码
4. **分块发送**：每 1920 samples (120ms) 发送一次，与服务端 chunk_interval 配合
