# AI鼠标设备连接问题修复记录

## 问题描述

应用启动后，AI鼠标设备一直无法连接，`deviceId` 始终为 `null`。但在 `feature/ai-mouse-test` 分支上能正常连接。

## 问题分析

通过对比两个分支的代码差异，发现问题出在 `src/main/mouseSDK.js` 中的回调函数原型变量定义方式：

### 问题代码（当前分支）

```javascript
// 在 initSDK 函数内部用 const 定义局部变量
const OnDeviceConnectedCallbackPro = koffi.proto('void OnDeviceConnectedCallback(...)')
```

### 正确代码（feature/ai-mouse-test 分支）

```javascript
// 在模块级别用 let 定义
let OnDeviceConnectedCallbackPro = null

// 在 initSDK 中赋值
OnDeviceConnectedCallbackPro = koffi.proto('void OnDeviceConnectedCallback(...)')
```

## 问题原因

1. **回调原型变量作用域问题**：koffi 库需要回调原型变量保持在模块级别，以确保正确的生命周期管理。使用函数内部的 `const` 局部变量会导致 koffi 内部状态异常。

2. **SDK初始化顺序问题**：需要保持正确的初始化顺序：
   - 先 `SDK_sdkInit(debug)` 初始化 SDK
   - 然后 `getVoiceKey()` + `setVoiceKey(97)` 设置语音键
   - 最后 `registerCallbacks()` 注册回调函数

## 修复方案

### 1. 添加模块级回调原型变量

在 `src/main/mouseSDK.js` 文件顶部添加：

```javascript
// 回调函数原型（延迟初始化，必须是模块级变量）
let OnDeviceConnectedCallbackPro = null
let OnDeviceDisconnectedCallbackPro = null
let OnDeviceMessageCallbackPro = null
let OnDeviceAudioDataCallbackPro = null
```

### 2. 修改 initSDK 中的原型定义方式

将 `const` 改为直接赋值给模块级变量：

```javascript
// 注册设备连接回调（原型赋值给模块级变量）
OnDeviceConnectedCallbackPro = koffi.proto('void OnDeviceConnectedCallback(const char* deviceId, int connectionMode)')
SDK_registerDeviceConnectedListener = libm.func('registerDeviceConnectedCallbackListener', 'void', [koffi.pointer(OnDeviceConnectedCallbackPro)])

// 其他回调原型同样处理...
```

### 3. 保持正确的初始化顺序

```javascript
// 先初始化SDK
SDK_sdkInit(debug)

logger.info('MouseSDK', 'SDK初始化成功')

// 设置语音键（触发DLL内部状态刷新）
try {
  const currentVoiceKey = SDK_getVoiceKey()
  SDK_setVoiceKey(97)
} catch (error) {
  logger.warn('MouseSDK', '初始化时设置语音键失败', { error: error.message })
}

// 最后注册回调函数
registerCallbacks()
```

## 修改的文件

- `src/main/mouseSDK.js`

## 验证结果

- 快速重启（无等待）：设备连接成功 ✓
- 按键事件正常接收 ✓
- 设备ID正确识别：`A01AC128` ✓

## 相关经验

1. 使用 koffi 调用 DLL 时，回调原型变量必须定义在模块级别
2. DLL 在频繁重启后可能需要时间释放资源，建议增加轮询检测机制
3. SDK初始化后调用 `getVoiceKey()` 和 `setVoiceKey()` 可能触发 DLL 内部状态刷新
